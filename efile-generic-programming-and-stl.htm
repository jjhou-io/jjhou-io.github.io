這裡提供的是翻譯此書過程中檢閱的程式。主要由黃俊堯先生整理。感謝他。

每個 .cpp 都可獨立編譯，不需額外參數。以下並附一個 makefile for UNIX，名為 Makefile.gcc。

每個程式都以 g++ 測試過。g++ 的版本是：
    gcc version 2.95.2 19991024 (release)
在 Solaris/Linux/FreeBSD 都驗證過，唯以下兩個檔案有問題：
    11-02-01-1.cpp
    12-06-05-3.cpp
原因是 g++ 2.95.2 對於 const char[] 型別宣告上的處理有 bug。

由於這些程式都是在 Unix 系統上編輯，其 newline 符號不同於DOS/Windows 環境。如果你在 DOS/Windows 環境，可能有些文字編輯器（text editors）無法正確顯示格式。不過它們仍可不經任何修改直接交由 Win32 C++ 編譯器來處理。


所有程式都符合 C++ Standard。如果您在其他編譯器上無法編譯，應該是編譯器不符標準所致。關於Borland C++Builder、GCC、Microsoft Visual C++ 三者未能符合 C++ Standard 之處，我的個人經驗整理於
qa-cpp-primer-27.pdf    or    qa-cpp-primer-27.txt
