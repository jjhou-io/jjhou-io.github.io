本書被視為一個傳奇，
尤其在 MFC 被視為一個「古老」架構而本書仍銷售不輟的情況下。
當眾口鑠金「MFC早已落伍」，一本探討 MFC programming 的書籍
卻於 2001~2006 銷售近 90,000 冊，原因有兩個：
一是 MFC 實未消失於業界，還有不少應用軟體以它開發（並維護）。
二是本書其實探討 application framework — 那個令人畏懼+神往的層次。

我從來不是（也絲毫無意）做 MFC 代言人，更不會想
以一本書沿續一個老技術的生命（這只是讀者的謬讚，事實上不可能）。
MFC 並不多好，也不多糟，它就是軟件長河中的一個過程一個點而已。
一天到晚討論這個工具好那個工具差，棄此逐彼、棄彼逐此者，儘是 young men。
位列開發一線的程式員/工程師/專案主管，心態大不相同。

《深入淺出MFC2/e》繁體版用了將近 24 個月的時間，達到 10,000 本印量。

《深入淺出MFC2/e》簡體版用了不到 22 個月的時間，達到 50,000 本印量。

簡體版的潛在讀者較繁體版多出百十倍，我可以理解。但這樣的成績仍然令我感到十分驚訝和驚喜，原因有三：

1. 本書繁體版問世至今四年有餘。在這四年之中，Windows 應用軟件開發工具固然VC++/MFC仍領風騷，但百花爭豔。PC平台亦早已不復Windows獨霸局面。

2. 本書繁體版電子書已於2000/07免費開放（見本書繁體版網頁）。

3. 本書使用VC5/MFC4。時序進入2002年，VC7/MFC7已經面市。

這裡面說明了一些事實：

★ 這本探索MFC application framework內核架構（特別是與應用程序接軌主軸）的書，不受VC/MFC版本影響。

★ 本書技術主題、文字組織、寫作風格，受到讀者的歡迎。

身為作者的我，非常開心有機會在此表達個人情緒。

容我重申，99.99999% 的程序員終其一生不需要自行設計一個 application framework。儘管如此，本書呈現的MFC framework概念，可以帶給您多面向的幫助與技術水平的提昇。

99.99999% 的程序員學習VC/MFC，是為了應用。然而太多程序員在application framework的重重布幔之間跌倒。重重布幔使他們知其然卻不知所以然，見樹而未能見林，無法盡情發揮MFC的威力與自己的實力。

認識MFC底層機制，是過程，是手段。最終目的是為了良好運用MFC。本書揭示MFC的核心主軸，及與應用程序間的接軌介面，使你見樹又見林，使你知其然又知其所以然。隨著本書的進展，時而你會俯衝見樹，時而你會高躍見林。最後就是胸中自有丘壑。

祝君 胸有丘壑，揮灑自如。
