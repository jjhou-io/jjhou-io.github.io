本書附錄各範例程式之檢測記錄

《C++ Primer 3/e》厚達 75 頁的附錄，提供了 C++ Standard Library 的
所有 generic algorithms（泛型演算法）的說明與範例。對於想要學習
泛型程式設計的人，是一份很好的資源。整份程式碼可於
http://www.awl.com/cseng/titles/0-201-82470-1/ 下載而得。

我在製作《C++ Primer 3/e 中文版》時，曾隨機檢測了幾個範例，發現
在 VC6 和 BCB4 上不能有效通過編譯。由於 Standard Template Library（STL）
有眾多版本，各家編譯器對 template 的支援程度（以及對 C++ Standard 的
支援程度）亦不相同，所以我不宜直接在中文版更改原碼。

現在，我以手上的 VC6 和 BCB4 和 GNU C++ 為工具，對整個附錄的所有範例程式
做了檢測，並修改無法通過的部份。檢測記錄如下。

1. 將下載後的 code.zip 置於 c:\primer\bin
2. 解壓縮
3. 進入 c:\primer\bin\appendix，將所有 .C 改名為 .CPP

4. 以下是修改通則：

(a) 將表頭檔改為 C++ style。此雖不影響正誤，但可使程式風格一致。
<iostream.h> ==> <iostream>
<assert.h> ==> <cassert>
奇怪的是，在 BCB4 INCLUDE 子目錄中，所有的表頭檔案都是 .H 檔，
找不到 C++ style。但程式使用 C++ Style 表頭檔卻又不見有錯。why？
莫非其前置處理器將 #include <cassert> 自動改為 #include<cassert.h>？

(b) 在 #include<...> 之後，增加 using namespace std;
此法雖非最理想（會增加編譯時間），但卻最方便。
面對這些小型範例程式，增加的編譯時間微不足道！

(c) 程式中凡用到 allocator ，皆移除（於是使用預設引數）。

(d) 請在部份程式的 main() 結束前加上 return 0; 以消除編譯警訊。

★若出現以下 (e)(f)(g)(h)(i)(j)(k)(m) 情況，稍後程式實測記錄中會明示之★
(e) 程式中凡用到 set<string ...>，皆改為 set<string>，改用預設引數。
(f) 程式中凡用到 istream_iterator< string ... >，皆改為 istream_iterator< string >
(g) 部份程式應含入而未含入 <functional>
(h) 部份程式應含入而未含入 <iostream>
(i) 部份程式應含入而未含入 <string>
(j) count() 參數與回返值 (obsolete!)
(k) count_if() 參數與回返值 (obsolete!)
(m) times<int> 應改為 multiplies<int> (obsolete!)
註：在 HP STL 中原本命名為 times，因與 UNIX header <sys/times.h>
衝突，所以在 C++ Standard 中已改為 multiplies。
(n) 由於 BCB4 不支援只有單一引數（表示預設大小）的  list<int> ，所以
加上第二引數 -1（表示預設初值。注意，只可以是負數，怪）。
(o) 為特別應付 BCB4，將
list<int> ilist_result(ilist.size());
改為
int i = ilist.size();
list<int> ilist_result(i, -1);



5. 編譯器的使用：

(a) 使用 command line 模式，會比 IDE（整合開發環境）更方便。

★VC6 的環境設定如下（請依各人環境修改其中路徑）：
set PATH=C:\MSDEV\VC98\BIN;C:\MSDEV\COMMON\MSDEV98\BIN
set INCLUDE=C:\MSDEV\VC98\INCLUDE
set LIB=C:\MSDEV\VC98\LIB

★BCB4 的環境設定如下（請依各人環境修改其中路徑）：
set PATH=C:\inprise\CBuilder4\BIN
set INCLUDE=C:\inprise\CBuilder4\INCLUDE
set LIB=C:\inprise\CBuilder4\LIB

★GNU C++ (egcs-2.91.57) 的環境設定如下（請依各人環境修改其中路徑）：
set PATH=c:\CYGNUS\CYGWIN-b20\H-I586-cygwin32\BIN


(b) 在 Windows 中開啟 DOS box，設定好上述環境變數後，即可使用編譯器。

VC6 的用法：
cl -GX alg??.cpp <enter>

BCB4 的用法：
bcc32 alg??.cpp <enter>

GNU C++ (egcs-2.91.57) 的用法：
g++ -o alg??.exe alg??.cpp <enter>

就以上範例而言，三家編譯器的表現（非嚴謹測試, on intel machine, windows 98）：
編譯速度：VC6 優於 GCC(egcs-2.91.57) 優於 BCB4
EXE 大小：VC6 優於 BCB4 優於 GCC(egcs-2.91.57)
與 C++ STL相容性：三家都差不多

若論與 C++ Standard 的相容性，則根據我的其他經驗（非嚴謹測試）：
GCC(egcs-2.91.57) 優於 BCB4 優於 VC6
